Política de Cookies e Armazenamento Local
Versão 1.0 — Abril de 2026

1. Sobre esta Política

Esta política descreve como o Linger utiliza armazenamento local no dispositivo do usuário. Por ser um aplicativo mobile nativo, o Linger não utiliza cookies HTTP tradicionais em sua interface principal.

2. O que Armazenamos no Dispositivo

Armazenamento seguro (FlutterSecureStorage): Token de acesso JWT — necessário para autenticar requisições à API; token de refresh — necessário para renovar a sessão sem novo login; identificador do usuário (UUID) — necessário para identificar a sessão local. No Android, esses dados são armazenados em Encrypted Shared Preferences. No iOS, são armazenados no Keychain com acesso liberado após o primeiro desbloqueio do dispositivo.

Preferências do aplicativo (SharedPreferences): Flag de conclusão do onboarding — indica se o usuário já completou o processo de cadastro; flags de visualização da introdução do match — indicam se o usuário já viu o texto de apresentação de determinado match diário.

Armazenamento temporário em memória (RAM): O histórico do chat de onboarding é mantido apenas em memória durante o processo de cadastro e descartado ao fechar o aplicativo antes da conclusão. O código de indicação pendente e o identificador de verificação de telefone Firebase são mantidos em memória durante o fluxo de autenticação e descartados após a conclusão.

3. Firebase

O SDK do Firebase é utilizado para dois fins: verificação de número de telefone por SMS durante o login (Firebase Authentication) e envio de notificações push (Firebase Cloud Messaging). Durante o fluxo de verificação por SMS, o Firebase pode armazenar temporariamente dados de sessão no dispositivo, descartados após a autenticação ser concluída. Na versão web do aplicativo, o Firebase pode utilizar IndexedDB ou localStorage do navegador. Para detalhes, consulte a política de privacidade do Google em policies.google.com/privacy.

4. Finalidade

Todo o armazenamento local tem finalidade exclusivamente funcional: manter a sessão do usuário ativa, evitar login repetido e garantir o funcionamento das notificações. Não utilizamos armazenamento local para fins de rastreamento, publicidade ou análise comportamental.

5. Como Remover os Dados Locais

Os dados armazenados localmente são removidos automaticamente ao excluir a conta pelo aplicativo ou ao desinstalar o aplicativo do dispositivo. Você também pode limpar os dados manualmente nas configurações do seu dispositivo, em "Armazenamento do aplicativo".
